  <action>
    **usePlanetSelect.js** — Thin hook that wraps `useSceneDispatch` and returns a `useCallback` function accepting a planet `id`, dispatching `{ type: 'SELECT_PLANET', payload: id }`. Single import point for all planet-selection triggers.

    **NavSidebar.jsx** — Fixed sidebar on the left side of the screen (`fixed top-0 left-0 h-full w-56`).
    - Accepts `isOpen` prop (boolean) and `onClose` prop (function) from parent.
    - Lists `[SUN, ...PLANETS]` from `src/data/planets.js` as clickable buttons.
    - Each button shows `body.name` and a small circle swatch with `body.color` (inline `backgroundColor`).
    - Uses `usePlanetSelect` hook for click handler. After dispatching, calls `onClose()` to collapse the sidebar on mobile after selection.
    - Active highlight: reads `selectedPlanet` from `useSceneContext()` — if `selectedPlanet === body.id`, apply `bg-white/15` background.
    - Styling: `bg-black/80 backdrop-blur-sm`, `z-30`, `pointer-events-auto`, `overflow-y-auto`.
    - For now, apply `translate-x-0` always (responsive mobile toggle comes in Plan 04-02). The `isOpen` prop and `onClose` prop are accepted but mobile-specific toggle behavior is wired in 04-02.
    - Section header "SOLAR SYSTEM" in `text-white/30 text-[10px] uppercase tracking-widest font-semibold` at the top.
    - `transition-transform duration-300` for future mobile slide animation.

    **TimelineControl.jsx** — Absolutely positioned at the bottom center of the viewport.
    - `absolute bottom-4 left-1/2 -translate-x-1/2` with `pointer-events-auto`.
    - Contains a play/pause button and an `<input type="range">` slider.
    - Slider: `min={0} max={100} step={1}`, reads `value` from `useSceneStore((s) => s.speed)`.
    - `onChange` dual-dispatches: `dispatch({ type: 'SET_SPEED', payload: Number(e.target.value) })` to SceneContext AND `useSceneStore.getState().setSpeed(value)` to Zustand. This is the same pattern as `useKeyboardShortcuts` for TOGGLE_PAUSE.
    - Play/pause button: reads `isPaused` from `useSceneStore((s) => s.isPaused)`, dual-dispatches TOGGLE_PAUSE to both stores on click.
    - Display current speed as `{speed}x` label to the right of the slider.
    - Styling: `bg-black/60 backdrop-blur-sm rounded-full px-4 py-2 flex items-center gap-3`.
    - Slider: `w-40 accent-blue-400 cursor-pointer`.
    - Button text: `isPaused ? '▶' : '⏸'` (Unicode play/pause symbols).

    **Footer.jsx** — Fixed badge at bottom-right.
    - `fixed bottom-4 right-4 z-10 pointer-events-none`.
    - Text: "Created by AU Brussel" in `text-white/30 text-[10px] uppercase tracking-widest`.
    - Minimal styling — must not compete with the 3D scene.

    - Uses `useState(false)` for `animated` + `useEffect` that resets `animated` to `false`, then calls `requestAnimationFrame(() => setAnimated(true))` — this guarantees a paint at 0% width before the CSS transition fires.

    - Returns null if `!composition || composition.length === 0`.
    - Each entry renders: a label row (`name` + `percent%`) + a track div (`h-1.5 bg-white/10 rounded-full overflow-hidden`) + a fill div (`h-full bg-blue-400/70 rounded-full transition-[width] duration-700 ease-out`).
    - Fill div: `style={{ width: animated ? \`${percent}%\` : '0%' }}`.
    - IMPORTANT: Use `transition-[width]` NOT `transition` — Tailwind default transition does NOT include width.
