**1. Extend sceneStore.js** with a planetRef registry (3 new fields):

```js
planetRefs: {},  // { [planetId]: React.RefObject<THREE.Group> }
registerPlanetRef: (id, ref) => set(s => ({ planetRefs: { ...s.planetRefs, [id]: ref } })),
unregisterPlanetRef: (id) => set(s => {
  const { [id]: _, ...rest } = s.planetRefs
  return { planetRefs: rest }
}),
```

This lets CameraController look up any planet's groupRef by id to call `getWorldPosition`.

**2. Create `src/hooks/useCamera.js`** — exports `useCamera()` returning `{ flyTo, flyToOverview }`.

Key implementation details:
- Module-level scratch vector: `const _scratchVec = new Vector3()` — never allocate in callbacks
- `flyTo(planetData, groupRef)`:
  1. `groupRef.current.getWorldPosition(_scratchVec)` — captures world position at call time
  2. Compute camera offset: `{ x: tx, y: ty + offset, z: tz + offset }` where offset = `Math.max(planetData.radius * 8, 8)` — minimum distance 8 prevents camera going inside tiny planets (Mercury 0.35, Pluto 0.18)
  3. `controls.enabled = false` — CRITICAL: must disable BEFORE starting tween
  4. `gsap.killTweensOf(camera.position)` + `gsap.killTweensOf(controls.target)` — kill running tweens
  5. Tween A: `gsap.to(camera.position, { x: cx, y: cy, z: cz, duration: 1.5, ease: 'power2.inOut', onComplete: () => { controls.enabled = true } })`
  6. Tween B: `gsap.to(controls.target, { x: tx, y: ty, z: tz, duration: 1.5, ease: 'power2.inOut', onUpdate: () => controls.update() })` — onUpdate keeps controls internal state in sync
- `flyToOverview()`: Same pattern but targets `{ x: 0, y: 50, z: 120 }` (camera) and `{ x: 0, y: 0, z: 0 }` (target) — matches the Canvas camera.position in App.jsx
- Both functions accessed via `useThree(s => s.controls)` for the controls instance and `useThree(s => s.camera)` for camera
- Use `useCallback` with `[camera, controls]` deps

**3. Create `src/components/scene/CameraController.jsx`**:

- Renders `<OrbitControls ref={controlsRef} makeDefault enableDamping dampingFactor={0.05} minDistance={3} maxDistance={200} />`
- `makeDefault` registers controls in R3F root store (accessible via `useThree(s => s.controls)` anywhere)
- Uses `useCamera()` hook to get `flyTo` and `flyToOverview`
- `useEffect` watching `selectedPlanet` from `useSceneContext()`:
  - When `selectedPlanet` changes to a planet id: look up `useSceneStore.getState().planetRefs[selectedPlanet]` to get the groupRef, find planet data from PLANETS array, call `flyTo(planetData, groupRef)`
  - When `selectedPlanet` changes to null: call `flyToOverview()`
- Also calls `useKeyboardShortcuts()` (from Task 2) to activate keyboard listeners

IMPORTANT anti-patterns to avoid:
- Do NOT use `useSceneContext()` inside `useFrame` — causes re-renders. The useEffect pattern is correct here because selectedPlanet changes infrequently (only on user interaction).
- Do NOT re-enable OrbitControls in `onUpdate` — only in `onComplete`
- Do NOT animate only camera.position without animating controls.target — camera snaps back when OrbitControls re-enables
