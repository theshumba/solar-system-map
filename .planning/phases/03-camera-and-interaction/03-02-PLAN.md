**2. Create `src/components/scene/HoverLabel.jsx`**:

- Uses `<Html>` from `@react-three/drei` with `center` and `distanceFactor={60}` props
- `position={[0, data.radius * 1.6, 0]}` — offset above planet surface. Accept `radius` as a prop (not `data`), so the interface is: `<HoverLabel name={data.name} radius={data.radius} />`
- Styling: dark translucent background (`bg-black/60` or inline `rgba(0,0,0,0.55)`), white text, `text-xs font-semibold tracking-widest uppercase`, `pointer-events: none`, `user-select: none`, `white-space: nowrap`, `border-radius: 4px`, `padding: 2px 8px`
- Do NOT use `occlude` — unnecessary complexity for a hover-only label (only one visible at a time)
- Do NOT use `transform` mode — amplifies positional lag at high orbital speeds

**1. Create `src/components/ui/InfoPanel.jsx`** — persistent DOM panel, never unmounts:

Structure:
- Always mounted in the DOM (persistent render pattern — never conditional render)
- `translate-x-full` when hidden (off-screen right), `translate-x-0` when visible
- `transition-transform duration-300 ease-in-out` for smooth slide-in/out
- `pointer-events-auto` to re-enable interaction on the panel (parent DOM overlay is pointer-events-none)
- `overflow-y-auto` for scrollable content on small screens

Implementation:
- Read `selectedPlanet` from `useSceneContext()` and `dispatch` from `useSceneDispatch()`
- Build a `PLANET_MAP` lookup: `Object.fromEntries([...PLANETS, SUN].map(p => [p.id, p]))` — module-level, computed once
- `const planet = selectedPlanet ? PLANET_MAP[selectedPlanet] : null`
- `const isOpen = !!planet`

Content layout (when `planet` is truthy):
1. **Close button** — absolute top-right, `X` icon, dispatches `{ type: 'SELECT_PLANET', payload: null }` on click. This is required for mobile/keyboard-less users (per research open question #3).
2. **Planet name** — `<h2>` with `text-2xl font-bold text-white`
3. **Nickname/tagline** — `<p>` with `text-sm text-gray-400` below the name
4. **6 key stats** — iterate `Object.entries(planet.stats)`, render as flex rows with label (gray) on left and value (white) on right. Format camelCase keys with regex: `key.replace(/([A-Z])/g, ' $1').trim()` then capitalize first letter.
5. **Fun facts carousel** — `useState(factIndex)` starting at 0. `useEffect` with `setInterval` every 4 seconds cycling `factIndex = (factIndex + 1) % planet.funFacts.length`. Reset `factIndex` to 0 when `selectedPlanet` changes (separate `useEffect([selectedPlanet])`). Clear interval on cleanup. Render inside a styled box (`bg-white/5 rounded-lg p-4 text-sm italic min-h-[80px]`). Show a small dot indicator below the fact text showing which fact is active.
6. **Notable moons** — if `planet.notableMoons?.length > 0`, render a section with "Notable Moons" header and a comma-separated list of moon names. If no notable moons, show only the moon count from stats.

Styling:
- Panel width: `w-80` (320px) on all screens
- Background: `bg-black/80 backdrop-blur-sm`
- Positioned: `fixed right-0 top-0 h-full` (or `absolute` if inside the overlay div)
- Z-index: `z-20` (above the Canvas overlay root which is z-10)
- Inner padding: `p-6`
- Section spacing: `space-y-4` or manual `mt-4` margins

IMPORTANT design choices:
- InfoPanel is ALWAYS mounted — content updates via props, visibility via CSS translate-x. This preserves scroll position across planet switches and avoids animation restarts.

- Fun facts interval is cleared when planet changes or component unmounts — no stale intervals.
- `pointer-events-auto` on the panel itself re-enables mouse interaction; the parent overlay remains `pointer-events-none` so clicks pass through to the Canvas.
