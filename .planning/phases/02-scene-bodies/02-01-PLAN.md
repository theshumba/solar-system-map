**3. Create Sun component** (`src/components/scene/Sun.jsx`):
- Import `SUN` from `../../data/planets`
- Import `useTexture` from `@react-three/drei`
- Load `SUN.texture` via `useTexture`
- Render `<mesh>` with `<sphereGeometry args={[SUN.radius, 64, 64]}>` at position `[0, 0, 0]`
- Material: `<meshStandardMaterial map={texture} emissive={SUN.emissiveColor} emissiveIntensity={SUN.emissiveIntensity} emissiveMap={texture} />`
- Add `<pointLight>` as sibling to mesh (NOT child): `intensity={2}`, `decay={0}`, `position={[0, 0, 0]}`, `color="#ffffff"`
- `decay={0}` ensures light reaches all planets regardless of distance (scene-scale PointLight)
- No orbital animation — Sun stays at origin

**4. Create Planet component** (`src/components/scene/Planet.jsx`):
- Props: `data` (one entry from PLANETS array)
- Import `useRef` from `react`, `useFrame` from `@react-three/fiber`, `useTexture` from `@react-three/drei`
- Import `useSceneStore` from `../../store/sceneStore`
- Load texture: `const texture = useTexture(data.texture)`
- Refs: `const groupRef = useRef()` (orbital group), `const meshRef = useRef()` (planet mesh)
- Pre-compute constants outside useFrame (but inside component — they depend on props):
  - `ORBITAL_RATE = (2 * Math.PI) / (data.orbitalPeriod * 365.25)` — radians per Earth-day
  - `SELF_ROT_RATE = (2 * Math.PI) / (Math.abs(data.rotationPeriod) * 86400)` — radians per second (rotation period is in Earth days, convert to seconds for delta-based animation)
  - `ROT_SIGN = data.rotationPeriod < 0 ? -1 : 1` — Venus and Uranus/Pluto have retrograde rotation
- Accumulated angle ref: `const angleRef = useRef(Math.random() * Math.PI * 2)` — random start position so planets don't all start at the same angle
- `useFrame((_state, delta) => { ... })`:
  - Read `{ speed, isPaused } = useSceneStore.getState()` — NO React re-render
  - If not paused: `angleRef.current += delta * ORBITAL_RATE * speed`
  - Set position: `groupRef.current.position.set(Math.cos(angleRef.current) * data.distance, 0, Math.sin(angleRef.current) * data.distance)`
  - If not paused: `meshRef.current.rotation.y += ROT_SIGN * delta * SELF_ROT_RATE * speed`
- JSX structure (two-level for clean tilt separation):
  ```jsx
  <group ref={groupRef}>
    <mesh ref={meshRef} rotation={[0, 0, (data.axialTilt * Math.PI) / 180]}>
      <sphereGeometry args={[data.radius, 32, 32]} />
      <meshStandardMaterial map={texture} />
    </mesh>
    {/* Plan 02-02 will add children here for moons, rings, clouds */}
  </group>
  ```
- The group handles orbital positioning; the mesh handles axial tilt and self-rotation
- Export as default function `Planet`

**5. Create OrbitLine component** (`src/components/scene/OrbitLine.jsx`):
- Props: `radius`, `color` (default `'#334455'`), `opacity` (default `0.3`)
- Import `useMemo` from `react`, `Line` from `@react-three/drei`
- `SEGMENTS = 128`
- `points = useMemo(() => { ... }, [radius])` — generate array of `[cos(angle)*radius, 0.01, sin(angle)*radius]` for 0..SEGMENTS (closed loop: i <= SEGMENTS)
- Return `<Line points={points} color={color} lineWidth={0.5} transparent opacity={opacity} depthTest={false} />`
- `depthTest={false}` and Y=0.01 together prevent z-fighting (per research)
