**2. Create SaturnRings component** (`src/components/scene/SaturnRings.jsx`):
- Props: `innerRadius`, `outerRadius`
- Import `useRef`, `useMemo` from `react`; `RingGeometry`, `Vector3` from `three`; `useTexture` from `@react-three/drei`
- Import `TEXTURE_PATHS` from `../../data/constants`
- Load ring texture: `useTexture(TEXTURE_PATHS.saturnRing)`
- Create geometry with UV remap in `useMemo` (critical — default RingGeometry UVs are wrong for radial textures):
  ```js
  const geometry = useMemo(() => {
    const geo = new RingGeometry(innerRadius, outerRadius, 128, 1)
    const pos = geo.attributes.position
    const uv = geo.attributes.uv
    const v3 = new Vector3()
    for (let i = 0; i < pos.count; i++) {
      v3.fromBufferAttribute(pos, i)
      const normalized = (v3.length() - innerRadius) / (outerRadius - innerRadius)
      uv.setXY(i, normalized, 1)
    }
    return geo
  }, [innerRadius, outerRadius])
  ```
- Render:
  ```jsx
  <mesh geometry={geometry} rotation-x={Math.PI / 2}>
    <meshBasicMaterial
      map={ringTexture}
      side={2}            // THREE.DoubleSide
      transparent={true}
      depthWrite={false}  // CRITICAL — prevents z-fighting with Saturn sphere
      opacity={0.9}
    />
  </mesh>
  ```
- `rotation-x={Math.PI / 2}` rotates ring from XY to XZ plane (flat horizontal)
- `depthWrite={false}` is mandatory per research (Pitfall 5)

**2. Create AsteroidBelt component** (`src/components/scene/AsteroidBelt.jsx`):
- Import `useRef`, `useEffect`, `useMemo` from `react`; `useFrame` from `@react-three/fiber`; `Object3D` from `three`
- Import `useSceneStore` from `../../store/sceneStore`; `ORBITAL_DISTANCES` from `../../data/constants`
- Constants:
  ```js
  const COUNT = 2000
  const INNER = ORBITAL_DISTANCES.mars + 2    // just outside Mars (16 + 2 = 18)
  const OUTER = ORBITAL_DISTANCES.jupiter - 3 // just inside Jupiter (30 - 3 = 27)
  const scratch = new Object3D()              // PRE-ALLOCATED outside component — zero per-frame allocation
  ```
- `scratch` MUST be declared outside the component (module level) — this is the critical performance rule
- Pre-generate instance data in `useMemo`:
  ```js
  const instanceData = useMemo(() => {
    const data = []
    for (let i = 0; i < COUNT; i++) {
      data.push({
        radius: INNER + Math.random() * (OUTER - INNER),
        angle: Math.random() * Math.PI * 2,
        y: (Math.random() - 0.5) * 1.5,
        speed: 0.02 + Math.random() * 0.03,
        scale: 0.05 + Math.random() * 0.1,
        rotX: Math.random() * Math.PI,
        rotY: Math.random() * Math.PI,
      })
    }
    return data
  }, [])
  ```
- Initialize matrices in `useEffect`:
  ```js
  useEffect(() => {
    if (!meshRef.current) return
    instanceData.forEach((d, i) => {
      scratch.position.set(Math.cos(d.angle) * d.radius, d.y, Math.sin(d.angle) * d.radius)
      scratch.rotation.set(d.rotX, d.rotY, 0)
      scratch.scale.setScalar(d.scale)
      scratch.updateMatrix()
      meshRef.current.setMatrixAt(i, scratch.matrix)
    })
    meshRef.current.instanceMatrix.needsUpdate = true
  }, [instanceData])
  ```
- `useFrame` animation:
  ```js
  useFrame((_state, delta) => {
    if (!meshRef.current) return
    const { speed, isPaused } = useSceneStore.getState()
    if (isPaused) return

    instanceData.forEach((d, i) => {
      d.angle += delta * d.speed * speed
      scratch.position.set(Math.cos(d.angle) * d.radius, d.y, Math.sin(d.angle) * d.radius)
      scratch.rotation.set(d.rotX, d.rotY, 0)
      scratch.scale.setScalar(d.scale)
      scratch.updateMatrix()
      meshRef.current.setMatrixAt(i, scratch.matrix)
    })
    meshRef.current.instanceMatrix.needsUpdate = true
  })
  ```
- JSX:
  ```jsx
  <instancedMesh ref={meshRef} args={[null, null, COUNT]} frustumCulled={false}>
    <dodecahedronGeometry args={[0.07, 0]} />
    <meshStandardMaterial color="#888888" roughness={0.9} metalness={0.1} />
  </instancedMesh>
  ```
- `frustumCulled={false}` prevents the entire belt from disappearing when camera looks away (Pitfall 2)
