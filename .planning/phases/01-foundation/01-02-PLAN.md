    2. Create `src/components/scene/Starfield.jsx`:

    A simple wrapper around drei's `<Stars>` component. This is procedural (no texture loading), so it goes OUTSIDE the Suspense boundary in the Canvas.

    ```jsx
    import { Stars } from '@react-three/drei'

    export default function Starfield() {
      return (
        <Stars
          radius={200}
          depth={60}
          count={8000}
          factor={4}
          saturation={0}
          fade
          speed={0.5}
        />
      )
    }
    ```

    Props explained:
    - `radius={200}` — star sphere covers the entire scene (Neptune at ~64 units)
    - `count={8000}` — dense enough to feel immersive but not GPU-heavy on mobile
    - `saturation={0}` — white/grey stars (correct for space)
    - `fade` — stars fade toward camera horizon for depth
    - `speed={0.5}` — subtle twinkling, not distracting

    3. Create `src/components/scene/PostProcessing.jsx`:

    ```jsx
    import { EffectComposer, Bloom, Vignette } from '@react-three/postprocessing'

    export default function PostProcessing() {
      return (
        <EffectComposer>
          <Bloom
            luminanceThreshold={0.9}
            luminanceSmoothing={0.025}
            mipmapBlur
            intensity={1.5}
            radius={0.85}
          />
          <Vignette
            offset={0.3}
            darkness={0.6}
            eskil={false}
          />
        </EffectComposer>
      )
    }
    ```

    CRITICAL: `luminanceThreshold={0.9}` — this is the Phase 1 lock. Only materials with emissiveIntensity >= 2.0 (the Sun, added in Phase 2) will trigger bloom. Stars, planets, orbit lines — none of them will accidentally glow. Do NOT change this value.

    4. Update `src/App.jsx` to wire everything together:

    ```jsx
    import { Canvas } from '@react-three/fiber'
    import { Preload } from '@react-three/drei'
    import { Suspense } from 'react'
    import { SceneProvider } from './context/SceneContext'
    import LoadingScreen from './components/ui/LoadingScreen'
    import Starfield from './components/scene/Starfield'
    import PostProcessing from './components/scene/PostProcessing'

    export default function App() {
      return (
        <SceneProvider>
          <div className="relative w-screen h-screen bg-black overflow-hidden">
            {/* 3D WORLD */}
            <Canvas
              className="absolute inset-0"
              dpr={[1, 2]}
              /* shadows intentionally disabled — PointLight shadow maps prohibitive at solar system scale */
              gl={{ antialias: true, alpha: false, powerPreference: 'high-performance' }}
              camera={{ position: [0, 50, 120], fov: 60, near: 0.1, far: 10000 }}
              style={{ touchAction: 'none' }}
            >
              {/* Stars: OUTSIDE Suspense — procedural, no loading needed */}
              <Starfield />

              <Suspense fallback={null}>
                {/* SolarSystem component added in Phase 2 */}
                <PostProcessing />
                <Preload all />
              </Suspense>
            </Canvas>

            {/* 2D WORLD — DOM overlays positioned over Canvas */}
            <LoadingScreen />
          </div>
        </SceneProvider>
      )
    }
    ```

    Architecture notes:
    - `Starfield` is OUTSIDE `Suspense` — it's procedural (shader-based), no assets to load
    - `PostProcessing` is INSIDE `Suspense` — it needs the scene to be ready
    - `<Preload all />` triggers material pre-compilation
    - `LoadingScreen` is a DOM element OUTSIDE the Canvas, reading `useProgress` from drei
    - `Suspense fallback={null}` — Canvas shows nothing while loading (the DOM LoadingScreen handles the UI)
