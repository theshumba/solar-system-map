    - `TEXTURE_PATHS` object — paths for all textures (sun, mercury, venus, earth, earthClouds, mars, jupiter, saturn, saturnRing, uranus, neptune, pluto) pointing to `/textures/*.jpg` or `.png`

    2. Create `src/data/planets.js` with the PLANETS array and SUN object:

    The PLANETS array contains objects for Mercury, Venus, Earth, Mars, Jupiter, Saturn, Uranus, Neptune, and Pluto. Each object has ALL schema fields as defined in the research:
    - `id` (string) — lowercase planet name
    - `name` (string) — display name
    - `nickname` (string) — tagline (e.g., "The Swift Planet")
    - `radius` (number) — from PLANET_RADII in constants.js
    - `distance` (number) — from ORBITAL_DISTANCES in constants.js
    - `orbitalPeriod` (number) — in Earth years
    - `rotationPeriod` (number) — in Earth days
    - `axialTilt` (number) — in degrees
    - `texture` (string) — from TEXTURE_PATHS in constants.js
    - `color` (string) — fallback hex color
    - `emissive` (boolean) — false for all planets
    - `moons` (array) — empty for Phase 1 (populated in Phase 2)
    - `stats` object with: distanceFromSun, diameter, orbitalPeriod, dayLength, moonCount, avgTemperature — all strings with units, using real NASA data
    - `funFacts` array — 2-4 strings per planet with real science facts
    - `composition` array — objects with `{ name, percent }` for atmospheric/surface composition

    Import `ORBITAL_DISTANCES`, `PLANET_RADII`, `TEXTURE_PATHS` from constants.js. Reference them instead of hardcoding values.

    Export the SUN separately:
    ```javascript
    export const SUN = {
      id: 'sun', name: 'The Sun', radius: PLANET_RADII.sun, distance: 0,
      emissive: true, emissiveColor: '#FDB813', emissiveIntensity: 2.0,
      stats: { ... }, funFacts: [...], composition: [...]
    }
    ```

    Use REAL NASA data for all stats, fun facts, and compositions. This data is used in the info panel in Phase 3.

    3. Create `src/context/SceneContext.jsx`:

    Use `createContext` + `useReducer` pattern. State shape:
    - `selectedPlanet: null` (string | null)
    - `hoveredPlanet: null` (string | null)
    - `speed: 1` (number, 0-100)
    - `isPaused: false` (boolean)
    - `cameraMode: 'overview'` ('overview' | 'focused')
    - `isLoaded: false` (boolean)

    Reducer handles: SELECT_PLANET, HOVER_PLANET, SET_SPEED, TOGGLE_PAUSE, SET_LOADED.

    Export: `SceneProvider`, `useSceneContext`, `useSceneDispatch`.

    Use two separate contexts (one for state, one for dispatch) to avoid unnecessary re-renders when only dispatch is needed.

    4. Create `src/App.jsx` with the Canvas/DOM composition:

    ```jsx
    import { Canvas } from '@react-three/fiber'
    import { SceneProvider } from './context/SceneContext'

    export default function App() {
      return (
        <SceneProvider>
          <div className="relative w-screen h-screen bg-black overflow-hidden">
            {/* 3D WORLD */}
            <Canvas
              className="absolute inset-0"
              dpr={[1, 2]}
              /* shadows intentionally disabled — PointLight shadow maps prohibitive at solar system scale */
              gl={{ antialias: true, alpha: false, powerPreference: 'high-performance' }}
              camera={{ position: [0, 50, 120], fov: 60, near: 0.1, far: 10000 }}
              style={{ touchAction: 'none' }}
            >
              {/* Starfield, Suspense, PostProcessing added in Plan 01-02 */}
            </Canvas>

            {/* 2D WORLD — DOM overlays positioned over Canvas */}
            {/* LoadingScreen added in Plan 01-02 */}
          </div>
        </SceneProvider>
      )
    }
    ```

    This establishes the architecture: SceneProvider wraps everything, Canvas fills the viewport, DOM elements positioned absolutely on top.
